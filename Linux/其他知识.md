#### 挂载

```shell
lsblk # 查看挂载设备 通常设备在/dev目录下

mount /dev/sdb1 /mnt/usb # 挂载到指定目录下

umount /dev/sdb1 # 取消挂载
```



#### 添加静态路由

```shell
ip route add <目标网络>/<子网掩码> via <网关地址> dev <网络接口> # 添加静态路由

ip route del <目标网络>/<子网掩码> # 删除静态路由

ip route show # 查看当前路由表

route -n # 查看当前路由表, 直接显示ip地址
```



#### 计算机网络

##### 交换机

局域网内部通信：

​	•	交换机工作在OSI模型的**第二层（数据链路层）**，它的主要功能是基于MAC地址转发数据帧。

​	•	交换机用于连接同一局域网（LAN）中的设备，使这些设备能够在同一个网络段（或子网）内进行通信。

​	•	在局域网内部，交换机通过查看MAC地址表，将数据帧从一个设备转发到另一个设备，从而实现设备之间的互联。



##### 路由器

跨网络通信：

​	•	路由器工作在OSI模型的**第三层（网络层）**，主要负责基于IP地址进行数据包的路由选择。

​	•	路由器用于连接不同的子网或网络段，管理不同网络之间的数据流动。它能够分析IP地址并决定数据包应该发送到哪个网络。

​	•	路由器还可以通过NAT（网络地址转换）实现内部网络与外部网络（如互联网）之间的通信。

网络隔离和管理：

​	•	路由器能够隔离不同的子网，使得不同的网络段之间可以独立管理和控制。通过路由器，可以在不同网络之间设置访问规则和流量控制策略。



##### 实例

​	部门A：使用 192.168.1.0/24 网段的IP地址。

​	部门B：使用 192.168.2.0/24 网段的IP地址。

​	路由器：连接并管理这两个子网，路由器的两个接口分别配置为 192.168.1.1 和 192.168.2.1。



网络配置

​	•	部门A的设备：

​	•	计算机1：192.168.1.10

​	•	打印机：192.168.1.20

​	•	部门B的设备：

​	•	计算机2：192.168.2.10

​	•	服务器：192.168.2.20

​	•	**路由器**：

​	•	接口1：192.168.1.1（连接部门A的子网）

​	•	接口2：192.168.2.1（连接部门B的子网）

​	•	**交换机**：

​	•	交换机连接所有部门的设备以及路由器的接口，用于在物理层面实现设备的互联。



通信流程

​	1.	同一子网内的通信（部门A）：

​	•	部门A的计算机1 (192.168.1.10) 想要打印文件，它会直接将数据包发送到打印机 (192.168.1.20)。

​	•	由于它们属于同一个网段（192.168.1.0/24），数据包直接通过交换机传输，不需要经过路由器。

​	2.	不同子网之间的通信：

​	•	部门B的计算机2 (192.168.2.10) 想要访问部门A的打印机 (192.168.1.20)。

​	•	计算机2会发现目标IP地址 192.168.1.20 不在它的子网内（192.168.2.0/24），于是它会将数据包发送到它的默认网关，即路由器的 192.168.2.1 接口。

​	•	路由器接收到数据包后，识别出目标IP 192.168.1.20 属于另一个子网（192.168.1.0/24），于是将数据包转发到它的 192.168.1.1 接口，然后通过交换机将数据包传输到财务A的打印机。

​	•	打印机收到数据包后，处理请求并将响应数据通过相同路径返回给部门B的计算机2。
